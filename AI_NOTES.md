# AI Notes

AI helped write most of AniSwipe end-to-end, especially the Node/Express backend, MongoDB/Mongoose models, seed scripts, React views, swipe-card UI, results, matches, undo, demo vote seeding, and the basic analytics dashboard. Codex was the main coding assistant for implementation, file edits, backend routes, frontend components, CSS polish, and verification commands. ChatGPT helped earlier with interpreting the assignment and planning the architecture before implementation.

The main design decisions I made were choosing anime as the theme, using MongoDB as the database, using lightweight username login instead of anonymous `sessionId`, and enforcing vote deduplication with a MongoDB unique compound index on `{ userId, itemId }`. I also decided to rename the app to AniSwipe, keep Jikan only in the backend seed script, treat MongoDB as the source of truth after seeding, filter out non-show anime like music videos, use visible tabs for navigation, include an `end` test user, seed demo users for populated results, and target a polished 390x844 mobile layout.

I pushed back most on frontend behavior and visual polish. One concrete issue was the swipe card logic: it looked like the card disappeared instead of swiping away. I looked into it myself and realized the animation was set so fast that the card was leaving almost instantly, so the timings had to be rewritten. I also pushed back on image dragging, missing swipe hints, and the anime background, where the implementation did not match what I was seeing in the browser. The frontend implementation was worse than I expected in those moments because Codex was not always clear on my bug prompts. In contrast, I found the backend design to be very well done and organized without any issues whatsoever.

One thing AI did better than expected was structuring the backend cleanly: routes, models, validation, upsert voting, aggregate results, matches, seed scripts, and analytics all fit together in a way that was easy to explain. One thing AI did worse than expected was interpreting visual bugs from screenshots, especially around card animations and page background opacity.

The AI tools used were ChatGPT for assignment interpretation and architecture planning, and Codex for implementation, debugging, terminal commands, code edits, README updates, and final cleanup. I verified the final app myself by manually testing login, seeding, voting, duplicate vote behavior, undo, results, matches, analytics, and the mobile layout. When I found issues on the UI side, I called them out and iterated until they were fixed.
